Project Descriptions

Data Enclaves for Scientific/Secure Computing (DESC) is a new architecture appropriate to the needs of applications for which existing commercial trusted execution environments (TEEs) have shortcomings. These applications include the following.

High Performance Computing

This project is developing new architectures appropriate to the performance and usage needs of scientific computing to secure scientific data from the edge to the HPC center. Scientific data today is at risk due to how it is collected, stored, and analyzed in highly disparate computing systems. We believe that in order to solve the problems described above that future HPC hardware and software solutions should be co-designed together with security and scientific computing integrity concepts designed and built into as much of the stack from the outset as possible.

Edge Computing

This project is developing new architectures appropriate to the performance and usage needs of scientific computing to secure scientific data from the edge to the HPC center. This includes includes sensor and edge systems that collect and process of that data takes place outside protection boundaries of traditional HPC centers, including against attacks such as ransomware and physical attacks against the computing system. Our approach will address the gaps left by existing solutions for scientific workflows to address the specific power, performance, and usability, and needs from the edge to the HPC center.

The secure HPC and Edge computing projects are supported by the US Department of Energy’s Office of Science’s Advanced Scientific Computing Research (ASCR) program under the following grants:

“Toward a Hardware/Software Co-Design Framework for Ensuring the Integrity of Exascale Scientific Data,” PI: Sean Peisert, 2015.

“Cybersecurity for Edge-to-Center Scientific Computing in Advanced Wireless Environments,” PI: Sean Peisert, Co-PIs: Venkatesh Akella and Jason Lowe-Power, 2021.

Nuclear Treaty Verification

It is essential to have the ability to ensure that software operating on arms control monitoring equipment has not undergone modification and associated data has not been disclosed. It has been many years since most treaties and other agreements were implemented. Existing approaches to ensuring data confidentiality, and software and data integrity, are outdated and thus are easily defeated by a motivated adversary. We are co-designing hardware/software security mechanisms that will allow a “secure computational data enclave continuum” — end-to-end security of nuclear monitoring data from the sensor through to the analysis and detection, not only at rest, but during computation.

This project is funded by NNSA’s Office of Defense Nuclear Nonproliferation (DNN):

“Data Enclaves for Secure Computing (DESC): Enabling Secure Nuclear Treaty Verification in Hostile Environments,” PI: Sean Peisert, Co-PIs: Venkatesh Akella, David Archer, and Jason Lowe-Power, 2024.

DESC is also funded by LBNL Contractor Supported Research.

For more information on DESC, see also the hardware-software security page at DArchR, the UC Davis Architecture Research group.
